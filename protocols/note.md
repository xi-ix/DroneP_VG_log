# 指标解释（外接校准头训练与检测评估）

## 训练过程中的两个参数

### 1) cls_acc
- 含义：校准头在当前 epoch 的二分类准确率。
- 二分类任务：判断候选框是“应保留(1)”还是“应丢弃(0)”。
- 计算：`cls_acc = 预测正确的样本数 / 样本总数`。
- 直觉：值越高，说明模型越能区分有效框和无效框。

### 2) pred_pos_rate
- 含义：当前 epoch 中，被模型判为正类（应保留）的候选框比例。
- 计算：`pred_pos_rate = 判为正类的样本数 / 样本总数`。
- 直觉：
	- 数值偏高：模型更激进，会保留更多框（可能提高召回，也可能引入更多误检）。
	- 数值偏低：模型更保守，会保留更少框（可能减少误检，也可能增加漏检）。

## 检测评估中的三个参数

### 3) Acc@0.5 
- 含义：IoU 阈值为 0.5 时的匹配准确率（本项目实现里更接近“按 GT 统计的召回比例”）。
- 规则：预测框与同类别 GT 框 IoU >= 0.5 视为匹配成功。
- 计算：`Acc@0.5 = 成功匹配的 GT 框数 / GT 框总数`。

### 4) Acc@0.75 
- 含义：IoU 阈值为 0.75 时的匹配准确率。
- 规则：预测框与同类别 GT 框 IoU >= 0.75 才算成功，要求比 0.5 更严格。
- 计算：`Acc@0.75 = 成功匹配的 GT 框数 / GT 框总数`。
- 说明：通常会低于 Acc@0.5，用于反映定位精度是否更“紧”。

### 5) mAP@0.5
- 含义：在 IoU=0.5 条件下，按类别计算 AP 后取平均得到 mAP。
- AP（Average Precision）反映 Precision-Recall 曲线下面积；mAP 是各类别 AP 的均值。
- 作用：相比单一 Acc 指标，mAP 同时考虑排序质量、召回和精度，更全面评估检测性能。
