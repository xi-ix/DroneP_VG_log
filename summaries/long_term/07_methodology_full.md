# 长期总结-项目方法总览（详细版）

本文档系统梳理 DroneP_VG 当前阶段使用的方法体系，覆盖数据准备、推理策略、评估协议、轻量训练、主模型微调、对照实验与复现规范。目标是让后续论文撰写、答辩展示和新实验接续可直接复用。

## 1. 任务定义与研究目标

### 1.1 任务定义
- 场景：VisDrone 航拍密集小目标检测。
- 目标：在不进行大规模端到端重训练的约束下，提高 class-aware 检测质量。
- 评价对象：100 图评估子集（GT boxes=5077）。

### 1.2 优化目标
- 主目标：提升 `Acc@0.5`、`Acc@0.75`、`mAP@0.5`。
- 工程目标：控制推理耗时与显存，保持流程可复现。
- 研究目标：识别“候选框覆盖”和“候选排序/校准”两类瓶颈的相对贡献。

### 1.3 约束条件
- 主干模型以 GroundingDINO 零样本能力为基础。
- 优先采用“低侵入改造”：阈值/滑窗/融合/外接轻量模块。
- 保持统一评估口径，防止跨实验结果不可比。

## 2. 数据与标注处理方法

### 2.1 数据子集组织
- 评估主集：`visdrone_test_100/`（100 图）。
- 报告子集：`visdrone_report_subset_120/`（100 正样本 + 20 负样本）。

### 2.2 标注读取与兼容
- 同时兼容空格格式与 VisDrone 原生逗号格式。
- 在评估脚本中跳过非法行并记录 `skipped` 数量，确保统计口径透明。

### 2.3 切片与映射
- 使用 `scripts/import_cv2.py` 做滑窗切片。
- 方法要点：
  - 原图坐标到切片坐标的仿射映射。
  - 支持空切片保留（便于负样本分析）。
  - 输出 metadata 与 summary，支持后续可追溯分析。

### 2.4 数据画像
- 使用 `scripts/analyze_extracted_dataset.py` 生成分布图、样例图与日志。
- 核心发现：小目标占比高（约 61.32%），是方法设计必须显式考虑的难点。

## 3. 推理主线方法（GroundingDINO）

### 3.1 基线推理
- 脚本：`scripts/run_groundingdino_baseline.py`。
- 可控参数：`box_threshold`、`text_threshold`、`nms_threshold`、`max_detections`、`device`。
- 输出：每图预测文件（可被评估脚本直接读取）。

### 3.2 阈值搜索
- 脚本：`scripts/search_groundingdino_thresholds.py`。
- 目的：在不改模型结构前提下，通过后处理参数提升 class-aware 指标。
- 机制：自动遍历参数网格，逐组执行推理与评估，并汇总为 Markdown/TSV。
- 当前 best 整图配置：`box=0.20, text=0.20, nms=0.40`。

### 3.3 滑窗策略
- 核心思路：将高分辨率图像切片后做局部推理，再回映射到原图坐标系。
- 网格搜索参数：`window in {896,1024,1280}` × `overlap in {0.10,0.20,0.30}`。
- 已观察到“Acc 导向最优”和“mAP 导向最优”配置分化，说明排序质量与召回并非同步单调。

### 3.4 结果融合（Stage A 非训练版）
- 输入：整图 best 预测 + 滑窗 best 预测。
- 融合策略：按类合并候选框并执行 class-aware NMS。
- 作用：在不训练的条件下提供稳定增益，作为后续训练版对照底座。

## 4. 评估方法与指标定义

### 4.1 统一评估脚本
- 脚本：`scripts/evaluate_detection_metrics.py`。
- 输出：统一格式的 `Detection Evaluation Summary`。

### 4.2 主要指标
- `Acc@0.5`：IoU>=0.5 的 GT 匹配比例（项目实现语义接近 GT 召回）。
- `Acc@0.75`：IoU>=0.75 的 GT 匹配比例（更强调定位质量）。
- `mAP@0.5`：按类别计算 AP 后求平均。
- `Empty-image false positive rate`：空图误报率。

### 4.3 评估口径
- 默认采用 class-aware（类别需匹配）。
- 保留 class-unaware 对照，用于拆分“类别错配”与“框定位”两类误差来源。

## 5. 轻量训练方法（外接校准头）

### 5.1 设计动机
- 在“不修改或少修改主干”的前提下，提高候选框保留决策质量。
- 将问题从“重训检测器”转化为“候选框校准/重排”。

### 5.2 Stage A 外接模块（SL）
- 脚本：`scripts/train_external_calibrator_stageA.py`。
- 输入特征：来自整图与滑窗候选的框级信息。
- 模型：轻量 MLP 校准头（输出保留概率）。
- 训练目标：二分类监督学习（保留/丢弃候选框）。
- 后处理：搜索 score threshold + NMS IoU。

### 5.3 双阶段训练（SL + RL）
- 在监督学习收敛后，使用策略梯度做 RL 微调。
- RL 目标：优化候选保留策略，使全图检测指标代理值提升。
- 稳定化机制：
  - early stopping；
  - best checkpoint 回载；
  - 受限阈值/NMS 搜索空间，避免退化到极端参数。

### 5.4 训练可观测性
- 每 epoch 记录训练/验证指标。
- 实时落盘 TSV 进度文件，支持中断恢复与可视化复核。

## 6. 主模型微调方法（可行性与增强）

### 6.1 可行性验证
- 脚本：`scripts/finetune_groundingdino_feasibility.py`。
- 使用小样本、少 epoch 快速验证反向更新可执行性。

### 6.2 微调范围策略
- `head`：只开检测头相关参数。
- `head_plus`：在 head 基础上再开放 encoder/feat_map 的少量参数。

### 6.3 与外接模块联动
- 先用微调后的主模型生成候选预测。
- 再接入 SL+RL 外接模块做同口径对照。
- 目标：验证“提升候选质量 + 候选重排”是否优于“仅候选重排”。

## 7. 跨模型对照方法（Qwen 系列）

### 7.1 对照目的
- 验证通用视觉语言模型在当前检测协议下的适配性。

### 7.2 执行方式
- 脚本：`scripts/run_qwen3vl_detection_baseline.py`。
- 与 GroundingDINO 主线使用同评估集、同评估脚本，保证横向可比。

### 7.3 作用定位
- 不作为当前主线优化对象。
- 主要用于路线论证：说明当前任务中 GroundingDINO 主线更具可行性。

## 8. 运行效率评估方法

### 8.1 基准对象
- `full_best`、`sw_cfg02`、`sw_cfg07`。

### 8.2 统计指标
- 总耗时（100 图）。
- 单图平均耗时。
- 峰值显存（MB）。

### 8.3 方法意义
- 将“精度收益”与“部署成本”同时纳入决策。
- 为后续选型提供 Pareto 参考，而非仅比较单一精度指标。

## 9. 文档化与复现方法

### 9.1 统一记录协议
- 按 `project_records/protocols/progress_save_protocol.md` 维护：
  - 会话日志；
  - 统一实验报告；
  - 会话交接；
  - 必要时更新长期总结。

### 9.2 结果组织原则
- 指标文档与原始预测分目录管理。
- 文件命名含方法与日期，便于追溯。
- 在报告中同时给出“指标值 + 文件路径”。

### 9.3 当前推荐复现实验链路
1. 运行整图基线并校验评估输出。
2. 做阈值搜索得到整图 best。
3. 执行滑窗网格并选择目标配置（Acc 或 mAP 导向）。
4. 执行 Stage A 融合（非训练）作为稳健增强底线。
5. 需要进一步提分时，执行 SL/SL+RL 外接训练。
6. 若上限受限，转向主模型小样本微调 + 外接联动验证。

## 10. 现阶段方法学结论

- 项目已形成“候选生成-候选重排-统一评估-文档归档”的完整方法体系。
- 仅依赖候选重排存在上限，后续提分应优先关注候选覆盖质量提升。
- 方法选择应采用三目标联合标准：`Acc@0.5`、`mAP@0.5`、运行成本（耗时/显存）。
- 当前可直接用于答辩主线的方法叙述为：
  - GroundingDINO 零样本主干；
  - 阈值搜索 + 滑窗增强 + Stage A 融合；
  - 外接轻量校准（SL+RL）；
  - 小样本主模型微调联动验证。
